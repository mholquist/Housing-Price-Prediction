There are several approaches we can take to dealing with missing values. However, some columns have over 50% missing values, and theres not much we can do with that. Even when there are greater than 20% missing values and you try to impute or remove the values, this changes the structure of the data too much. Therfore, I need to drop these columns with greater than 50% missing, and then decide what to do with the other columns.

That being said, one column in particular that stood out among the columns with > 20% missing values is yr_remodel. After checking the unique values, I think it is safe to assume that if the remodel year is missing it means it did not recieve a remodel. Not all houses are going to be remodeled and it is a small enough proportion to assume this is the case. I will therefore fill the missing remodel year with the build year (it got remodeled the same time it was built theoretically).

Splitting Into X and Y¶

Our target variable is TOTAL_VALUE since we are trying to predict the value of a property. Therefore, this is our y.

For our X set, There are a few columns we do not need at all:

  • Owner and owners mail address both serve as identifiers for the property. We need the model to learn how to predict home value without these clues, therefore we will exclude these variables from our X set.
  • LU_DESC is a description of the land use, but we already get this from the LU variable, so it is repetitive to include both.

Additionally, some columns function as identifiers, so we will need to remove these in order to have the models properly learn how to predict property values. The columns that function as id's are: CM_ID, GIS_ID, ST_NUM, ST_NAME, BLDG_SEQ, OWNER, and OWNER_MAIL_ADDRESS

Lastly, some variables may be a direct function of our y variable, so I cannot use them in our x set. These variables are land and building value.

I will include all other variables in the X set.
